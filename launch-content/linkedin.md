## 📝 Post

```
I've been building something in my spare time that I think could genuinely change how developers learn.

It's called CodeTube — and it's an open source platform for interactive coding tutorials.

Here's the problem it solves:

You find a great coding tutorial on YouTube. You watch it on 2x speed. The instructor writes some code. You want to try a variation. So you pause, open your editor, copy the current state of the code, try your idea, and then you've lost your place entirely.

This loop is inefficient. More importantly, it discourages experimentation — which is how real learning happens.

---

**CodeTube takes a different approach:**

Instead of recording video, tutorials are captured as *editor event streams* — keystrokes, cursor movements, file operations. Playback happens inside Monaco Editor (the same engine that powers VS Code).

As a viewer, you can:
→ Watch the code appear character by character in a real editor
→ Pause at any point and start editing the code yourself
→ Fork the tutorial state — your changes live in a branch alongside the instructor's version
→ Hit "Reset to Instructor" to restore the original code instantly
→ Jump to checkpoints — pre-defined states the instructor drops throughout the tutorial

---

**Why open source?**

The paid alternatives in this space work, but they're closed ecosystems. Companies can't self-host them for internal developer onboarding. Educators can't run their own instances. Developers can't contribute improvements.

I believe the right infrastructure for developer education should be open. MIT licensed. Self-hostable.

---

**What I've built so far:**

The backend is complete — authentication, tutorial management, recording upload/streaming, progress tracking, the branching system, and WebSocket-based real-time session management. The recording format spec is finalized. The frontend architecture is designed. Now I'm working on the playback engine.

This has been months of early mornings and weekends. The kind of project that reminds you why you got into software in the first place.

---

```
